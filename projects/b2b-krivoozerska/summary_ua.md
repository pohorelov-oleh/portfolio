# 📦 KStore – Кабінет Замовлень Дистриб'ютора

## 📌 Основна інформація
- **Розробник:** Погорелов О.Є.
- **Рік розробки:** 2024
- **Стек технологій:** ASP.NET Core, Microsoft SQL Server, Entity Framework Core, Microsoft Graph API, DataTables
- **Призначення:** Автоматизація отримання замовлень від дистриб'юторів, листування, контроль статусів замовлень, погодження документів та звітність.

## 🚀 Огляд проєкту
До впровадження **KStore**, замовлення приймалися вручну через електронну пошту, що спричиняло втрати даних та ускладнювало контроль виконання.  
**KStore** дозволяє дистриб'юторам легко створювати замовлення, стежити за їхнім статусом та взаємодіяти з виробником у єдиному цифровому просторі.

## 📌 Ключові можливості
✔ **Динамічний список замовлень** із фільтрами  
✔ **Автоматичне сповіщення** про зміну статусу замовлення  
✔ **Вбудований модуль листування** між дистриб'юторами та виробником  
✔ **Погодження документів** із автоматичним нанесенням QR-кодів  
✔ **Інтерактивні звіти** та експорт даних у `.xlsx`  
✔ **Гнучка система користувачів** з розмежуванням доступу

## 📌 Основні функціональні розділи

### 📝 1. Листування
- Внутрішня система обміну повідомленнями між дистриб'юторами та виробником  
- **Функціонал:**  
  - Тема листа, важливість, вкладення  
  - Розширений текстовий редактор **TinyMCE**  
  - Фільтри: `"Отримані"`, `"Надіслані"`, `"Новий лист"`

### 📋 2. Замовлення
- Відображення та управління всіма замовленнями  
- **Фільтри:** номер замовлення, дата, дистриб'ютор, місто, статус  
- **Статуси:** `"Виконано"`, `"Відхилено"`, `"В процесі виконання"`  
- **Автоматичне сповіщення** через **Microsoft Graph API**  
- **Калькулятор ваги** та піддонів для контролю максимального завантаження  

### 📄 3. Документи
- Завантаження **PDF-документів**  
- **Функціонал:**  
  - Погодження або відправлення на доопрацювання  
  - Автоматичне нанесення **QR-коду**  
  - Генерація штрих-коду для швидкого пошуку документів  

### 💰 4. Прайс-листи
- Створення **індивідуальних прайс-листів** для кожного дистриб'ютора  
- **Редагування тільки для адміністратора**  
- Автоматичне призначення цін при реєстрації нового дистриб'ютора  

### 📞 5. Контакти
- Каталог контактних осіб виробника  
- **Функціонал:**  
  - Додавання, редагування та видалення контактів **(тільки адміністратор)**  
  - Фільтрація за посадою  
  - Відображення номерів телефонів та email  

### 📊 6. Звіти
- Формування звітів **за день** або **за період**  
- **Експорт у `.xlsx`** для подальшого імпорту в 1С  
- Автоматична фільтрація та сортування даних  

### 🚀 7. Статуси замовлень
- Відображення замовлень у реальному часі  
- **Розділи:**  
  - `"На опрацюванні"`  
  - `"Відвантажено"`  
  - `"Відхилено"`  
- Використання **DataTables** для швидкого пошуку та сортування  

### 🔔 8. Оголошення
- Модуль оголошень для дистриб'юторів  
- **Функціонал:**  
  - Додавання зображень  
  - Текстовий редактор  
  - Дати публікації  
  - Можливість вмикати та вимикати оголошення  

### 👤 9. Користувачі
- Реєстрація **адміністратором**  
- **Налаштування ролей** і прав доступу  
- Підтримка ролей **Distributor, Manager, Administrator**

## 🏗 Архітектура проєкту
**KStore** побудований за архітектурою **MVC (Model-View-Controller)**, що забезпечує масштабованість і зручність у підтримці.

📂 KStore Solution
├── 📁 Controllers – API-контролери
├── 📁 Models – моделі бази даних
├── 📁 Views – представлення UI
├── 📁 Services – бізнес-логіка
└── 📁 wwwroot – стилі, скрипти, статичні файли
